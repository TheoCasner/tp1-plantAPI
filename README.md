# Usage

## Introduction 

This API provides simple routes helping you to get informations on plants !

## Swagger

Routes will be briefly explained below, but I invite you to check the swagger by launching the API.
You can find the swagger at `/docs`

## Routes

### Default

GET `/` -> Find All Plants

GET `/content` -> Route to check if you are old enough to not have parent control, requires a base64 encoded userId as authorization Header

### Plant

GET `/plant/:id` -> Find One PlantA

GET `/plant/search/:prefix` -> Find All Plants Matching Prefix

### Admin

POST `admin/user` -> add a new user, requires a base64 encoded userId as authorization Header

PUT `admin/user` -> update a user role, requires a base64 encoded userId as authorization Header

## Authorization

You can use the swagger or use Postman and add following header -> **authorization**
